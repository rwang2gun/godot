# 입력 매핑 설계서

**버전**: v0.1 (설계만, 구현은 패드부터)
**작성 일자**: 2026-05-08
**대상 환경**: ROG Ally X (Xbox 패드 내장 + 터치) + 옵션으로 키보드·마우스
**참조**: `LEMMINGS_CONTROLS_REFERENCE.md` (원작 UI 패턴 분석)

---

## 1. 기획 의도

### 1.1 목적
세 가지 입력 디바이스(**패드 / 터치 / 키보드+마우스**) 모두에 대해 **일관된 게임 액션 → 디바이스 입력 매핑**을 정의한다. 디바이스가 바뀌어도 같은 게임 행동이 가능해야 하며, 각 디바이스의 강점을 반영한다.

### 1.2 우선순위 (구현 순서)
1. **Pad** — 작업 환경(ROG Ally X)에서 기본 입력. v1.0 필수
2. **Touch** — Ally X 터치스크린. v1.1 권장
3. **Keyboard + Mouse** — 도크 연결 시. v1.2 — 데스크톱 PC 이식 시 필수

### 1.3 설계 원칙
- **액션 레벨 추상화** — 디바이스마다 분기하지 않고 액션(예: `skill_select_next`, `cursor_move_to`)으로 통일
- **패드 우선** — Lemmings 류는 마우스 친화적이지만 본 환경에서 마우스 없음. 패드가 일급 시민
- **자동 보정** — 패드는 정밀 클릭 어려우므로 군중 클릭 정확도 보정 필요
- **일관성** — 모든 디바이스에서 동일한 게임 진행 흐름. 어느 디바이스로 시작해도 막힘 없이 가능

## 2. 게임 액션 사전

플레이어가 게임에서 할 수 있는 모든 행동을 디바이스 무관 액션으로 정의. **input/keymap.gd에 이 카탈로그 그대로 등록**.

### 2.1 카메라 / 시야

| 액션 ID | 의미 | 발동 빈도 |
|---|---|---|
| `camera_pan` | 카메라 이동 (8방향) | 매우 높음 |
| `camera_zoom_in` | 줌 인 | 중간 |
| `camera_zoom_out` | 줌 아웃 | 중간 |
| `camera_focus_cursor` | 커서/선택 위치로 카메라 점프 | 낮음 |
| `minimap_toggle` | 미니맵 표시 토글 | 낮음 |

### 2.2 커서 / 포인터

| 액션 ID | 의미 | 비고 |
|---|---|---|
| `cursor_move` | 커서 위치 이동 | 패드/마우스/터치 |
| `cursor_target_next_ant` | 가까운 다음 개미로 커서 점프 | **패드 정확도 보정** |
| `cursor_target_prev_ant` | 가까운 이전 개미로 커서 점프 | 패드 정확도 보정 |

### 2.3 스킬 선택

| 액션 ID | 의미 | 비고 |
|---|---|---|
| `skill_select_n` (1~8) | n번 슬롯 직접 선택 | 키보드/숫자 단축 |
| `skill_select_next` | 다음 슬롯 사이클 | 패드 LB/RB |
| `skill_select_prev` | 이전 슬롯 사이클 | 패드 LB/RB |
| `skill_assign` | 현재 커서 위치 개미에 선택 스킬 부여 | 핵심 액션 |

### 2.4 게임 컨트롤

| 액션 ID | 의미 | 비고 |
|---|---|---|
| `pause_toggle` | 일시정지 / 재개 | |
| `release_rate_up` | 스폰 레이트 증가 | |
| `release_rate_down` | 스폰 레이트 감소 | |
| `speed_up` | 빨리감기 | 시뮬 tick 가속 |
| `speed_normal` | 정상 속도 | |
| `nuke` | 전체 자폭 (포기) | 더블 입력 또는 홀드 권장 |
| `restart_stage` | 스테이지 재시작 | 더블 입력 권장 |
| `back_menu` | 메뉴로 나가기 | |

### 2.5 보조

| 액션 ID | 의미 | 비고 |
|---|---|---|
| `info_toggle` | 현재 커서 위치 개미 정보 표시 | 길게 누르기 |
| `cursor_priority_toggle` | "다음 레밍 우선" 토글 (NeoLemmix 패턴) | |

## 3. 디바이스별 매핑

### 3.1 패드 (Xbox 배열) — **v1.0 구현 우선**

ROG Ally X 내장 패드는 Xbox 배열. 표준 Xbox 컨트롤러와 동일.

| 입력 | 액션 | 비고 |
|---|---|---|
| **좌 스틱** | `cursor_move` (가상 커서 이동) | 가속 곡선 적용 |
| **우 스틱** | `camera_pan` | 8방향 |
| **D-Pad ↑/↓** | `release_rate_up` / `release_rate_down` | |
| **D-Pad ←/→** | `cursor_target_prev/next_ant` | **개미 스냅 점프** (패드 정확도 보정) |
| **A** | `skill_assign` | 커서 위치 개미에 부여 |
| **B** | `back_menu` (단발) / `restart_stage` (홀드 1초) | 더블 액션 |
| **X** | `info_toggle` (단발) — 커서 아래 개미 상세 | |
| **Y** | `cursor_priority_toggle` | NeoLemmix 패턴 |
| **LB** | `skill_select_prev` | 슬롯 사이클 |
| **RB** | `skill_select_next` | 슬롯 사이클 |
| **LT** | `camera_zoom_out` (홀드) | 아날로그 |
| **RT** | `camera_zoom_in` (홀드) | 아날로그 |
| **View(좌)** | `pause_toggle` | |
| **Menu(우)** | `minimap_toggle` (단발) / `nuke` (홀드 2초) | 더블 액션 |
| **L3 (좌스틱 클릭)** | `camera_focus_cursor` | |
| **R3 (우스틱 클릭)** | `speed_up` (토글) | |

**핵심 패턴**:
- **좌 스틱 = 가상 커서**, **우 스틱 = 카메라** — 각자 독립
- **LB/RB로 스킬 사이클** — 8 슬롯을 좌우로 돌림 (PSP 패턴)
- **D-Pad ←/→로 개미 스냅 점프** — 군중 속에서 특정 개미 선택 정확도 보정. 본 입력이 패드 환경의 최대 가치
- **B/Menu는 단발/홀드 이중 액션** — 버튼 부족 보정

**가상 커서 동작**:
- 좌 스틱 입력 시 화면에 커서 표시
- 좌 스틱 정지 시 커서 위치 유지 (사라지지 않음)
- D-Pad ←/→로 점프 시 커서가 즉시 다음 개미 위치로 이동
- 카메라가 커서를 화면 가장자리로 밀면 자동 추적 활성

### 3.2 터치 (Ally X 터치스크린) — v1.1

화면 직접 터치 + 화면 하단 가상 패널 조합.

| 입력 | 액션 | 비고 |
|---|---|---|
| 화면 탭 (게임 영역) | `cursor_move` + `skill_assign` (즉시 부여) | 자동 줌 보정 후 |
| 화면 탭 (스킬 슬롯) | `skill_select_n` | 슬롯 직접 선택 |
| 두 손가락 핀치 | `camera_zoom_in/out` | |
| 두 손가락 드래그 | `camera_pan` | 한 손가락 드래그는 부여로 오해될 수 있음 |
| 길게 누르기 (개미) | `info_toggle` | |
| 길게 누르기 (스킬 슬롯) | 스킬 정보 |
| 화면 가장자리 드래그 | `camera_pan` | |
| 미니맵 영역 탭 | `camera_focus_cursor` | 즉시 점프 |
| 가상 버튼 (Pause/Rate/Nuke) | 각각 액션 | |

**자동 보정 (콘텐츠 기획서 §8.2와 정합)**:
- 군중 영역 탭 시 자동 줌 + 가장 가까운 개미 스냅
- 스킬 슬롯 탭 영역 확대 (실제 그래픽보다 큼)
- 의도 모호한 입력(2개미 사이) 시 우선순위 토글 따름

### 3.3 키보드 + 마우스 — v1.2 (도크 연결 또는 데스크톱 이식)

| 입력 | 액션 | 비고 |
|---|---|---|
| 마우스 이동 | `cursor_move` | |
| 좌클릭 | `skill_assign` | |
| 우클릭 | `cursor_priority_toggle` (모드) + 다음 레밍 우선 | NeoLemmix 패턴 |
| 마우스 모서리 | `camera_pan` (자동 스크롤) | 옵션 |
| 휠 업/다운 | `camera_zoom_in/out` | |
| 1~8 | `skill_select_n` | 직접 선택 |
| Q / E | `skill_select_prev/next` | 패드 LB/RB와 일관성 |
| WASD 또는 화살표 | `camera_pan` | |
| Space 또는 P | `pause_toggle` | |
| F1 / F2 | `release_rate_down/up` | 원작 호환 |
| F | `speed_up` (토글) | NeoLemmix 호환 |
| M | `minimap_toggle` | |
| R | `restart_stage` | 더블 입력 권장 |
| F12 | `nuke` | 더블 입력 권장 |
| Tab | `cursor_target_next_ant` | 패드 D-Pad와 일관 |
| Shift+Tab | `cursor_target_prev_ant` | |
| ESC | `back_menu` | |
| H 또는 길게 호버 | `info_toggle` | |

## 4. 입력 디바이스 우선순위 / 동시 사용

### 4.1 입력 모드 결정 로직

```
초기 상태: NONE
첫 입력 디바이스 감지 시 그 모드로 전환
입력 없이 5초 → 모든 모드 유효 (UI 힌트는 마지막 모드 기준)
디바이스 전환 시 즉시 UI 힌트 변경
```

UI 힌트 = 화면 표시되는 버튼 안내 (예: "A: 부여", "탭: 부여", "클릭: 부여").

### 4.2 동시 사용 (혼합 모드)

ROG Ally X 환경에서 자주 발생: 패드 + 터치 동시. 둘 다 활성 허용:
- 카메라 = 우 스틱 또는 두 손가락 드래그
- 스킬 부여 = A 버튼 또는 화면 탭
- **충돌 규칙**: 마지막 입력이 우선

## 5. UI / HUD 영향

### 5.1 디바이스별 UI 힌트

각 모드에서 화면에 표시할 단축키 힌트가 다름.

```
[Pad]   [LB] ← Skill → [RB]      [A] Assign
[Touch] Tap skill → Tap ant
[K+M]   1-8 Skill   Click Assign
```

### 5.2 가상 커서 표시 (패드 모드)

- 화면 중앙 또는 마지막 위치
- 좌 스틱 입력 시 표시
- 정지 후 3초 → 페이드 (단, 사라지지는 않음)
- 개미 스냅 시 짧은 애니메이션

### 5.3 스킬 슬롯 패널

세 모드 공통 — 화면 하단. 단, 모드별 보조 표시:

| 모드 | 추가 표시 |
|---|---|
| Pad | 좌측 [LB] / 우측 [RB] 아이콘. 현재 선택 슬롯 강조 |
| Touch | 슬롯 크기 ↑ (44pt 이상). 손가락 정확도 보정 |
| K+M | 각 슬롯에 숫자 1~8 작게 표시 |

### 5.4 미니맵

- 패드: Menu 버튼으로 토글
- 터치: 화면 우상단 작은 버튼
- K+M: M 키 토글

## 6. 구현 단계 (Pad 우선)

### Phase A — Pad 입력 기반 (v1.0 필수)
1. `input/keymap.gd`에 액션 사전(§2) 등록 + Project Settings InputMap 매핑
2. `input/pad.gd` 작성 — 패드 입력 → 액션 변환
3. **가상 커서 시스템** 구현 (`scripts/ui/virtual_cursor.gd`)
   - 좌 스틱 입력 → 화면 좌표 이동
   - 화면 가장자리 시 카메라 추적
   - 개미 스냅 점프 (`cursor_target_next/prev_ant`)
4. SkillToolbar에 LB/RB 사이클 + 슬롯 강조 시각
5. Pause / Rate / Nuke / Restart 핫키
6. UI 힌트 표시 (모드: Pad 고정)

### Phase B — Touch 추가 (v1.1)
1. `input/touch.gd` — 터치 → 액션
2. 자동 줌 보정 + 개미 스냅
3. 두 손가락 핀치 / 드래그
4. 가상 버튼 (Pause/Rate/Nuke 영역 확장)

### Phase C — Keyboard + Mouse 추가 (v1.2)
1. `input/desktop.gd` — 키보드 + 마우스 → 액션
2. 마우스 모서리 자동 스크롤 (옵션)
3. 우클릭 = `cursor_priority_toggle`
4. 단축키 풀 노출 (1~8, Q/E, F1/F2 등)
5. 사용자 키 리매핑 UI (옵션)

## 8. 리스크 / 결정 보류 사항

### 8.1 결정 필요
- [ ] **가상 커서가 없는 화면(메뉴/스테이지 선택)**에서 패드 동작 — 일반 UI 포커스 시스템으로 갈지 결정 필요
- [ ] **스킬 카운터가 0**인 슬롯도 사이클에서 보일지 — 보이되 회색 처리 권장
- [ ] **Tab/D-Pad 스냅 점프** 우선순위 규칙 — 화면 내 가까운 개미 / 화면 외 개미 포함 여부

### 8.2 알려진 함정
- **D-Pad ←/→가 너무 자주 입력**되면 군중 속 의도치 않은 선택 변경. 입력 쿨다운(0.1초) 권장
- **자동 카메라 추적**이 너무 공격적이면 멀미 유발. 화면 가장자리 도달 시점에만 작동
- **좌 스틱 가상 커서 가속 곡선**이 잘못 잡히면 정밀 조작 불가. v1.0에서 사용자 옵션으로 노출 권장 (Sensitivity 슬라이더)

### 8.3 모바일 이식 시 추가 사항
- 가상 패드(화면 좌하단 스틱 + 우하단 버튼)는 본 설계에 미포함 — 모바일 이식 시 별도 phase
- 터치 단독 모드는 작은 화면(<6인치) 시 자동 줌 더 적극적으로
