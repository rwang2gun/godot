# 레밍즈 조작 인터페이스 레퍼런스

**작성 일자**: 2026-05-08
**목적**: 레밍즈 시리즈의 UI·조작 패턴 조사 → CandyAnts UI 기획 참조 자료
**범위**: Lemmings(1991) 원작 + 주요 후속작·플랫폼별 변화 + 모던 이식

## 1. 원작 Lemmings (1991, DMA Design) — UI 패널 구조

화면 하단 고정 패널에 모든 조작이 집약. **마우스 중심 + 키보드 단축키 보조** 설계.

### 1.1 패널 레이아웃 (좌→우)

```
┌─────────────────────────────────────────────────────────────────────┐
│ [-] [+]  [CLIMB] [FLOAT] [BOMB] [BLOCK] [BUILD] [BASH] [MINE] [DIG] │
│ Rate     Skill 8개 (각 카운터 표시)                                 │
│                                                       [PAUSE] [NUKE] │
└─────────────────────────────────────────────────────────────────────┘
        [미니맵 — 전체 스테이지 축소도]
```

### 1.2 스킬 버튼 8종

각 버튼에 다음 정보 표시:
- **아이콘** — 스킬을 한눈에 식별 가능한 픽셀 아트
- **남은 횟수** — 사용 가능 카운트 (예: `4`, `12`)
- **선택 표시** — 현재 활성 스킬은 박스 강조

| 버튼 | 아이콘 모티프 | 단축키 (대부분 버전) | 비고 |
|---|---|---|---|
| Climber | 사다리 또는 위 화살표 | F3 또는 1 | 트레잇(영구) |
| Floater | 우산/낙하산 | F4 또는 2 | 트레잇(영구) |
| Bomber | 폭탄 또는 카운트다운 | F5 또는 3 | 액션 (5초 카운트) |
| Blocker | 정지 손 | F6 또는 4 | 액션 (영구 점유) |
| Builder | 계단 모양 | F7 또는 5 | 액션 (12계단 후 종료) |
| Basher | 수평 화살표 | F8 또는 6 | 액션 (수평 굴착) |
| Miner | 대각선 아래 화살표 | F9 또는 7 | 액션 (대각 굴착) |
| Digger | 수직 아래 화살표 | F10 또는 8 | 액션 (수직 굴착) |

### 1.3 게임 컨트롤 버튼 4종

| 버튼 | 기능 | 단축키 | 효과 |
|---|---|---|---|
| `[-]` Rate Down | 스폰 레이트 감소 | F1 또는 < | 출구에서 다음 레밍 나오기까지 텀 ↑ |
| `[+]` Rate Up | 스폰 레이트 증가 | F2 또는 > | 텀 ↓ (압박감 ↑) |
| `[Pause]` | 일시 정지 | P 또는 SPACE | 시뮬 정지. 카메라 스크롤·스킬 선택은 가능 |
| `[Nuke]` | 전체 자폭 | F12 (더블클릭) | 모든 살아있는 레밍에 Bomber 부여 — **포기 버튼** |

### 1.4 정보 표시 (HUD)

패널 상단 또는 측면:
- **OUT** — 현재 살아있는 레밍 수
- **IN** — 구출된 레밍 수 / 목표 (예: `IN: 12/20`)
- **TIME** — 남은 시간 (mm:ss)

### 1.5 마우스 조작

| 입력 | 동작 |
|---|---|
| 좌클릭 (스킬 버튼) | 스킬 선택 |
| 좌클릭 (레밍) | 선택된 스킬 부여 |
| 우클릭 | 스크롤 가속 (일부 버전) — 명시적 부여 액션 아님 |
| 마우스 화면 가장자리 | 카메라 자동 스크롤 |

### 1.6 키보드 단축키 (DOS 기준)

| 키 | 기능 |
|---|---|
| F1 / F2 | Release Rate 감소/증가 |
| F3~F10 | 스킬 1~8 선택 |
| F11 또는 P | Pause |
| F12 (더블탭) | Nuke |
| ESC | 메뉴 |
| 화살표 | 카메라 이동 |

### 1.7 미니맵

화면 최하단에 전체 스테이지 축소도. 레밍 위치를 점으로 표시. 클릭으로 즉시 카메라 이동.

## 2. 플랫폼별 변형 (1991~1995)

| 플랫폼 | 차이점 |
|---|---|
| Amiga (원작) | 마우스 + 키보드 단축키 풀 지원 |
| DOS PC | 키보드 단축키 F1~F12 매핑 |
| SNES | **D-pad + 버튼 조합**. 커서 이동 후 A로 부여. UI 패널 동일하지만 버튼 선택은 토글식 |
| Mega Drive | SNES와 유사한 D-pad 조작 |
| Game Boy | 작은 화면 → 미니맵 생략, 패널 축소 |
| 아케이드 | 트랙볼 + 버튼 |

→ **콘솔 이식의 핵심 과제: 마우스 정밀 클릭 → D-pad 커서 + 버튼 토글로 변환**. 이 패턴은 모바일 이식 시도 비슷한 문제 유발.

## 3. Lemmings 2: The Tribes (1993) — 60+ 스킬 UI

### 3.1 새 문제: 스킬 슬롯 폭증

원작 패널은 8 슬롯 고정이었으나 부족당 8 스킬 × 12 부족 = 96 스킬. 모두 노출 불가.

### 3.2 해결책: 스테이지별 8 스킬만 노출

각 스테이지가 **그 부족의 8 스킬 한 세트**만 사용. 패널 레이아웃은 원작 8 슬롯 유지하되 아이콘만 부족별로 변경.

### 3.3 부족 선택 UI

월드맵 화면에서 12 부족을 영역으로 표시. 영역 클릭 → 그 부족 스테이지 풀로 진입.

→ **CandyAnts 시사점**: 8 스킬 한 슬롯 패널은 인지 부하 적정선. 그 이상 노출하려면 부족·테마 단위로 분리 권장.

## 4. Lemmings (2006, PSP/DS) — 터치·휴대 시대

### 4.1 패널 변화

- 스킬 슬롯 8 유지하되 **버튼 크기 ↑** (터치 정확도 보정)
- 미니맵을 **상시 표시 영역**에서 토글 가능 영역으로 (좁은 화면)
- Release Rate를 슬라이더 → 두 버튼 (`-`, `+`) 으로 단순화

### 4.2 터치 상호작용 (DS)

| 입력 | 동작 |
|---|---|
| 스타일러스 탭 (스킬 버튼) | 스킬 선택 |
| 스타일러스 탭 (레밍) | 부여 |
| 두 손가락 핀치 | 줌 (일부 버전) |
| 화면 모서리 드래그 | 카메라 |

### 4.3 키 매핑 (PSP)

- L/R 버튼 → 스킬 슬롯 좌/우 사이클
- X → 부여
- Triangle → Pause
- Start → Nuke 메뉴

→ **CandyAnts 시사점**: PSP의 L/R 사이클 패턴이 게임패드/모바일 단축키 슬롯에 적용 가능

---

## 5. Lemmings Touch (2014, PS Vita) — 터치 전용

### 5.1 패널 → 화면 하단 단축 슬롯

원작 패널을 **간소화된 단축 바**로 축소. 8 스킬 중 일부만 슬롯에 배치.

### 5.2 제스처 도입

| 제스처 | 동작 |
|---|---|
| 탭 (스킬 슬롯) | 스킬 선택 |
| 탭 (레밍) | 부여 |
| 길게 누르기 | 스킬 정보 표시 |
| 두 손가락 스와이프 | 카메라 |
| 두 손가락 핀치 | 줌 |

### 5.3 자동 줌

레밍 탭 시도 시 주변 자동 확대 → 클릭 정확도 보정. **콘텐츠 기획서 §8.2의 "자동 줌인" 메카닉이 정확히 이 패턴**.

## 6. 후속작 NeoLemmix (커뮤니티 확장)

### 6.1 우클릭 = 우선순위

- 좌클릭: 통상 부여 (앞에 있는 레밍 우선)
- 우클릭: **다음 레밍 우선** — 군집 속에서 특정 레밍 선택 정확도 ↑

### 6.2 단축키 풀 노출

20+ 스킬 지원. 키보드 1~9 + Q~P 등으로 직접 매핑. 패널은 시각용, 실제 부여는 키보드 + 마우스.

### 6.3 시스템 단축키

| 키 | 기능 |
|---|---|
| ⏮️ Home | 카메라 시작점 |
| ⏭️ End | 카메라 끝점 |
| C | 클릭 우선순위 토글 |
| F | 카메라 따라가기 토글 |
| F2 | 빨리감기 |
| Backspace | 되감기 (NeoLemmix 신규) |

→ **CandyAnts 시사점**: 신규 패턴 두 가지가 흥미로움
- **클릭 우선순위 토글** (군집 클릭 정확도) — 모바일·복잡 레벨에서 가치 큼
- **되감기 (시간 역행)** — MVP 외로 큰 메카닉 추가지만 결정성 시뮬레이션과 자연스럽게 호환 (리플레이 시스템 응용)

## 7. 표준 UI 패턴 정리

원작~현대 모든 버전이 공유하는 **불변 패턴**:

| 영역 | 위치 | 필수 요소 |
|---|---|---|
| 스킬 슬롯 | 화면 하단 | 8개 (1세트), 각각 카운터 + 아이콘 |
| 게임 컨트롤 | 스킬 슬롯 좌우 끝 | Rate(±) + Pause + Nuke |
| HUD | 화면 상단 또는 패널 위 | OUT / IN / TIME / 목표 % |
| 미니맵 | 화면 하단 또는 토글 | 전체 스테이지 + 레밍 점 |
| 카메라 | 마우스 가장자리 / 드래그 | 자동 스크롤 + 미니맵 클릭 점프 |

### 7.1 정보 위계 (위에서 아래로 우선순위)

1. **즉시 보여야 함**: 현재 선택 스킬, 그 스킬 카운터, OUT/IN
2. **수시로 봐야 함**: TIME, Release Rate
3. **상황 시 봐야 함**: 미니맵, 다른 스킬 카운터
4. **메뉴에 숨겨도 됨**: 스테이지 목표 상세, 옵션

## 8. CandyAnts 적용 가이드

### 8.1 그대로 가져갈 수 있는 패턴

- 화면 하단 스킬 패널 8 슬롯 (Phase 3에서 SkillToolbar로 구현 예정)
- 각 슬롯에 카운터 표시
- Rate(-), Rate(+), Pause는 패널 양 끝
- HUD에 OUT(in_transit) / IN(saved) / lost / TIME 표시

### 8.2 CandyAnts 특화 변형

| 원작 | CandyAnts 권장 |
|---|---|
| OUT (살아있는 레밍 수) | **in_transit** (운반 중인 사탕 조각) — 4-카운터 ScoreSystem 기반 |
| IN (구출 수) | **saved** (귀환한 사탕 조각) |
| 별도 카운터 없음 | **lost** (영구 손실 사탕 조각) — CandyAnts 고유 |
| Nuke (전체 자폭) | **유지하되 의미 변경** — 운반 중 자폭 시 lost +N (포기 = 사탕 영구 손실 명시) |
| Release Rate | 동일 (개미 스폰 텀) |

### 8.3 모바일 이식 대비 (콘텐츠 기획서 §8.2와 정합)

- 자동 줌인 + 스냅 타겟 (Lemmings Touch 패턴)
- 단축키 슬롯 화면 하단 고정 (PSP 패턴)
- 두 손가락 핀치 줌 표준
- 시간 제한 +20% (이미 결정)

### 8.4 도입 검토 추천 패턴

| 원작 패턴 | CandyAnts 도입 여부 | 비고 |
|---|---|---|
| 8 슬롯 스킬 패널 | ✅ 필수 (Phase 3 SkillToolbar) | |
| Rate(±) + Pause + Nuke | ✅ 필수 | Nuke는 포기 메카닉 |
| 미니맵 | 🟡 v1.0 후 검토 | Phase 11 폴리싱 단계 |
| 키보드 단축키 1~8 | ✅ 권장 | PC 우선 설계 원칙 |
| 우클릭 = 다음 레밍 우선 | 🟡 검토 | NeoLemmix 패턴, 군중 정확도 ↑ |
| 자동 줌인 | 🔴 모바일 시 필수 | 모바일 이식 작업과 묶기 |
| 되감기 (NeoLemmix) | ⚫ MVP 외 | 결정성 시뮬과 호환되지만 별도 설계 필요 |

## 9. 단축키 표준안 (CandyAnts 권장)

```
스킬 선택:    1~8 (각 슬롯)
스폰 레이트: F1 (-), F2 (+)
일시정지:     P 또는 SPACE
빨리감기:     F2 (NeoLemmix와 동일 — 또는 다른 키)
재시작:       R
스테이지 종료: F12 (Nuke)
카메라:       방향키 또는 마우스 모서리
미니맵:       M (토글)
```

→ Phase 3 SkillToolbar 구현 시 keymap.gd 또는 InputMap에 사전 등록 권장.
