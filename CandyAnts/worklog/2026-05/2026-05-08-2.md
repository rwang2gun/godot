## 결정 / 변경

- **`.tdd_bypass` 토큰의 3중 가드 패턴 확립** — Phase 1 review의 MEDIUM 권고 기반. (a) `.gitignore`에 `scripts/hooks/.tdd_bypass` 패턴, (b) phase 완료 직전 `rm` + `git status --porcelain`으로 부재 확인, (c) deferred 금지(반드시 phase 내부 종결). 사유: 우회 토큰이 commit에 섞이면 후속 phase에서 TDD가 침묵 비활성화. 매 phase에 동일 패턴 재사용 — Phase 2도 동일 흐름.
- **headless 검증 시 `--fixed-fps 60` 필수** — `--quit-after`가 frames이긴 하나 wall-clock 의존 코드(Timer wait_time 등)와 어긋나 simulation 시간이 매우 짧게 흐르는 현상 발견. 30초 quit인데 첫 ant가 10초도 안 진행. `--fixed-fps 60`으로 강제 후 정상. 사유: ant_saved 기대값을 검증하려면 simulation/wall-clock 일치 필수. 향후 모든 phase 검증 명령에 적용.
- **Codex 리뷰 권고 — `Ant.has_candy: bool` 도입** — state(CarryingState)에서 candy 보유 여부를 분리. `effective_speed()`도 state가 아닌 has_candy 기반으로 전환해 Walker→Faller→Walker 사이클에도 0.78배 페널티 유지. 사유: Phase 5+(Hazard)와 Phase 6+(Basher의 절벽)에서 운반 중 떨어지는 시나리오 즉시 발현, Stage 1만 평지라 위장된 결함.
- **AntSpawner `_ensure_timer()` 패턴** — `_ready()`와 `start()` 양쪽에서 호출. 사유: Godot 형제 노드 `_ready()` 순서가 트리 구조 의존이라 fragile. Stage01.tscn에서 우연히 통과했지만 트리 변경에 취약. 향후 Spawner 재사용 시 안전.
- **CRLF 경고 다수** — Windows 환경. 동작 영향 없음. 명시적 수정 안 함.
